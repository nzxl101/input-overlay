input overlay
connection

Option #1 (recommended)

  1. get the input-overlay-ws server
  2. run it and right click the tray icon to copy your auth token
  3. paste the token in the auth token field below
  4. configure your overlay, then copy the URL and paste it as an OBS browser source
    • tip: edit config.json to change the address/port
    • tip: configure the key whitelist in the server settings
    🛈 Supports hall effect keyboards via the server settings.

Option #2

  1. install the input-overlay OBS plugin
  2. enable the websocket server in plugin settings
  3. restart OBS, copy the overlay URL, add it as a browser source
    4. 🛈 Easier setup but no authentication support.

key syntax

Comma-separated entries per row. Basic format:

[internal_name]:"[label]":[width class]

multi-key bindings

key_w|mouse_4:"W" - both W and mouse4 light up the same key
key_q|key_e:"use" - Q or E both trigger "use"

standard keys

label internal name
a-z key_a - key_z
0-9 key_0 - key_9
` - = [ ] \ ; ' , . / key_backtick, key_minus, key_equals…

special keys

label internal name
esc, tab, caps key_escape, key_tab, key_capslock
shift / ctrl / alt (L/R) key_leftshift, key_rightshift, etc.
space, enter, backspace key_space, key_enter, key_backspace
arrows key_leftarrow, key_uparrow, key_rightarrow, key_downarrow
f1-f12 key_f1 - key_f12
ins, del, home, end, pgup, pgdn key_insert, key_delete, key_home, key_end, key_pageup, key_pagedown
numpad 0-9 key_numpad_0 - key_numpad_9

mouse identifiers

label internal name
m1 (left) mouse_left
m2 (right) mouse_right
m3 (middle) mouse_middle
m4, m5 mouse_4, mouse_5
scroller scroller:"default":"up":"down"
side buttons mouse_side:"m5":"m4"

width classes

u0-01 through u7 in 0.01u increments

special syntax

syntax effect
invisible / key_escape:"invis":u2 invisible spacer (supports width classes)
dummy empty row placeholder
key_a|key_b:"A" multi-key binding via pipe
